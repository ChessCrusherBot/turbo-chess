# Turbo Chess

Turbo Chess is a Flutter chess training app with 30,000 drill positions and Play vs Computer mode.

The full license text is in [LICENSE](LICENSE). This project is prepared for GPLv3/open-source compliance from an engineering perspective because Turbo Chess bundles Stockfish, which is licensed under GPLv3. This is not legal advice or a legal guarantee.

## Stockfish

Turbo Chess includes Stockfish, licensed under GPLv3.

The Android app bundles native Stockfish binaries in:

- `android/app/src/main/jniLibs/arm64-v8a/libstockfish.so`
- `android/app/src/main/jniLibs/armeabi-v7a/libstockfish.so`
- `android/app/src/main/jniLibs/x86_64/libstockfish.so`

Recorded Stockfish source information is in:

- `assets/stockfish/STOCKFISH_SOURCE.txt`
- `lib/core/engine/BUILDING_STOCKFISH.md`
- `assets/legal/STOCKFISH_SOURCE.txt`
- `assets/legal/STOCKFISH_BUILDING.md`

## Third-Party Notices

Third-party notices are documented in [THIRD_PARTY_NOTICES.md](THIRD_PARTY_NOTICES.md). The Android app also bundles legal text assets under `assets/legal/` so users can access the same practical license/source information from the app package.

## Build

Install Flutter, then run:

```powershell
flutter pub get
flutter analyze
flutter test
flutter build apk --release
flutter build appbundle --release
```

## Android Release Notes

Play Store signing/configuration must be provided privately by the developer. Do not publish signing keys, keystore passwords, Play Console credentials, service account JSON files, developer payment/tax records, or private API keys.

The public source release should include the app source needed to build Turbo Chess, including Flutter/Dart source, Android project files, `pubspec.yaml`, `pubspec.lock`, tests, third-party notices, and Stockfish source/build metadata. Private secrets and signing credentials must remain outside the public repository.

`android/local.properties` is a local machine file generated by Flutter or Android Studio. It is intentionally ignored and should not be published; new developers can regenerate it by running Flutter/Android tooling locally.

## Secrets Warning

Never commit:

- private signing keys or keystore files
- `android/key.properties` or `android/keystore.properties`
- Play Console service account JSON files
- developer payment/tax records
- `.env` files
- private API keys

See [docs/SECRETS_AND_SIGNING.md](docs/SECRETS_AND_SIGNING.md) before publishing source publicly.
