# Signing and Private File Handling

This document explains how Turbo Chess handles signing and private local files in the public source repository.

## Signing keys

Android signing keys are private release material. They are excluded from this repository.

Developers who build Turbo Chess from source use their own local signing setup for release artifacts.

## Local Android signing config

Local signing configuration can be provided through files such as `android/keystore.properties` on a developer machine or in a private CI environment.

Those local configuration files are excluded because they can reference private signing keys or contain signing-related values.

Release signing depends on the developer's local configuration.

## Local machine files

`android/local.properties` is generated by Flutter or Android Studio for the local machine. It is excluded from version control and can be regenerated by Flutter tooling.

Build outputs, APK/AAB files, local caches, and release artifact folders are excluded from the public source tree.

## Play Console credentials

Play Console credentials, service account JSON files, upload-key material, Play App Signing records, and private developer account records are not included in this repository.

If a signing key or account credential is exposed, rotate it through the appropriate provider process.

## Current app state

The current Turbo Chess Android release is free, ad-free, offline-focused, and local-only. It does not include an ad SDK, subscriptions, in-app purchases, in-app payments, or Google Play Billing.

The Android release app does not request the `INTERNET` permission.
