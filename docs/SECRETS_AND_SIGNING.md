# Secrets And Signing

This document lists files and values that must stay private when Turbo Chess is released as open source.

## Do Not Commit

Never commit:

- keystore files such as `*.jks`, `*.keystore`, `*.p12`, or `*.pem`
- keystore passwords
- `android/key.properties`
- `android/keystore.properties`
- Play Console service account JSON files
- `service-account*.json`
- `*service_account*.json`
- `client_secret*.json`
- `.env` or `.env.*`
- private API keys
- AdMob account secrets
- bank/payment details
- private Play Console credentials

## Android Signing

Release signing should be done privately on the developer machine or in a private CI environment. The public repo can include build scripts that look for private signing configuration, but it must not include the private signing files or password values.

Turbo Chess currently reads release signing settings from `android/keystore.properties` when present. Keep that file private.

`android/local.properties` is generated by Flutter or Android Studio for the local machine. Keep it private and ignored. Other developers can regenerate it by running Flutter or opening the Android project locally.

## Play Store Upload Keys

Keep upload keys and Play App Signing credentials private. If a key is ever published, treat it as compromised and rotate it through the appropriate Play Console process.

## AdMob And Play Billing

AdMob app IDs, ad unit IDs, and Play Billing product IDs may appear in code when needed by the app, but do not publish account secrets, service account keys, private reports, payment information, or private API credentials.

## Firebase / google-services.json

`google-services.json` is not always a secret, but it can expose project configuration. If Turbo Chess adds Firebase later, review that file before deciding whether to publish it. Do not guess.

## Public Release Reminder

Before creating a public GitHub repository:

1. Run `git status`.
2. Run a secret scan.
3. Check `git ls-files` for secret-looking names.
4. Confirm `.gitignore` protects private files.
5. Keep signing, billing, and AdMob credentials outside the public repo.
