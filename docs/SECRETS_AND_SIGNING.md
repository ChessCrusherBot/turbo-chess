# Secrets and Signing

This document explains how Turbo Chess handles private signing and account material in a public source repository.

## Security and private files

This public repository intentionally does not include Android signing keys, keystore files, local machine configuration, Play Console credentials, environment files, private API keys, or private developer account records.

Those files are private to each developer or release environment. They should stay outside version control and should not be shared in public issues, pull requests, release notes, logs, screenshots, or build output.

## Android signing

Release builds may use local signing files when they are present on a developer machine or in a private CI environment. Turbo Chess can read release signing settings from `android/keystore.properties` when that file exists locally.

`android/keystore.properties` and similar signing configuration files are excluded from the public repository because they can point to private keys or contain signing-related values.

`android/local.properties` is generated by Flutter or Android Studio for the local machine. It is also excluded from version control. A developer can regenerate it by running Flutter tooling or opening the Android project locally.

Developers building from source should use their own local Android signing setup for release artifacts.

## Play Console credentials

Play Console credentials, service account files, upload keys, Play App Signing material, and private developer account records are not included in this repository.

If a signing key or credential is exposed, treat it as compromised and rotate it through the appropriate provider process.

## Current ads and payments status

The current Turbo Chess Android release is free, ad-free, offline-focused, and local-only. It does not include an ad SDK, active AdMob, subscriptions, in-app purchases, in-app payments, or Google Play Billing.

If monetization or cloud services are added in a future version, private account credentials and service keys should still remain outside the public repository.

## Maintainer checks

Before publishing a binary release or updating public source, maintainers should confirm:

- signing keys and keystore files are not tracked
- local machine configuration is not tracked
- Play Console credentials and service account files are not tracked
- environment files and private API keys are not tracked
- generated APK, AAB, and release output files are not tracked
